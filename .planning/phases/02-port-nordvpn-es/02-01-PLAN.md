<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Consumer workflow → nordvpn-es action | Untrusted: consumer provides username/password inputs |
| nordvpn-es action → NordVPN API | Trusted: fetching recommended server (read-only, public API) |
| nordvpn-es action → openvpn daemon | Trusted: local subprocess, config is bundled |
| nordvpn-es action → geo-IP providers | Untrusted: ipinfo.io and ifconfig.co are external; both must agree (NVES-07) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Username/password inputs | mitigate | Credentials written to `$RUNNER_TEMP/nordvpn-auth.txt` at mode 0600; removed by disconnect.sh; never echoed (NVES-11) |
| T-02-02 | Tampering | .ovpn config file | mitigate | Bundled in repo, SHA-pinned when consumer uses action; CA cert embedded prevents MITM |
| T-02-03 | Repudiation | N/A | accept | GitHub Actions provides audit log of workflow runs |
| T-02-04 | Information Disclosure | `set -x` in scripts | mitigate | NEVER use `set -x` (NVES-11, AGENTS.md § Security); `$GITHUB_STEP_SUMMARY` only shows safe diagnostics |
| T-02-05 | Denial of Service | NordVPN API failure | mitigate | API failure fails step with clear error (D-15: `::error::api.nordvpn.com returned empty hostname`); no silent success |
| T-02-06 | Elevation of Privilege | `sudo openvpn` | accept | Ubuntu runner requires sudo for openvpn; script runs with runner's privileges; no extra escalation |
| T-02-07 | DNS Leak | DNS queries bypass tun0 | mitigate | DNS-egress check in verify-country.sh (NVES-09): query via 127.0.0.53 (systemd-resolved); warn if not through tun0 |
| T-02-08 | Geo-IP Spoofing | Single provider returns wrong country | mitigate | Two-provider check: both ipinfo.io AND ifconfig.co must return ES (NVES-07); disagreement = hard fail |
| T-02-09 | Secret Leak in Logs | openvpn log contains auth | mitigate | `set -x` banned (NVES-11); disconnect.sh removes auth file; openvpn log goes to `$RUNNER_TEMP/openvpn.log` (runner-cleaned) |
</threat_model>
