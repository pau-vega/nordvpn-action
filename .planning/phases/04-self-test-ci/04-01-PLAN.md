**5b. self-test** — Matrix E2E test (TEST-03, TEST-04, TEST-05, TEST-06):
```yaml
  self-test:
    needs: fork-check
    if: needs.fork-check.outputs.is_fork == 'false'
    runs-on: ubuntu-latest
    environment: Preview
    strategy:
      matrix:
        region: [es, us, fr]
      fail-fast: false
    name: self-test (${{ matrix.region }})
    concurrency:
      group: self-test-${{ github.workflow }}-${{ matrix.region }}-${{ github.ref }}
      cancel-in-progress: false
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2

      - name: Connect NordVPN
        id: connect
        uses: ./actions/nordvpn-${{ matrix.region }}
        with:
          nordvpn-username: ${{ secrets.NORDVPN_SERVICE_USERNAME }}
          nordvpn-password: ${{ secrets.NORDVPN_SERVICE_PASSWORD }}

      - name: Post-connect verify
        env:
          REGION: ${{ matrix.region }}
          EXP_COUNTRY: ${{ matrix.region == 'es' && 'ES' || matrix.region == 'us' && 'US' || 'FR' }}
          EXIT_IP: ${{ steps.connect.outputs.exit-ip }}
          COUNTRY: ${{ steps.connect.outputs.country }}
          ASN: ${{ steps.connect.outputs.asn }}
          TUN0_STATE: ${{ steps.connect.outputs.tun0-state }}
          DEFAULT_ROUTE: ${{ steps.connect.outputs.default-route }}
          CONNECT_DURATION: ${{ steps.connect.outputs.connect-duration-ms }}
        run: |
          set -euo pipefail

          echo "=== Post-connect verification for nordvpn-${REGION} ==="

          # Assert all 6 outputs are non-empty (per D-15)
          for var in EXIT_IP COUNTRY ASN TUN0_STATE DEFAULT_ROUTE CONNECT_DURATION; do
            val="${!var}"
            if [ -z "$val" ]; then
              echo "::error::Output '${var,,}' is empty — action did not populate output"
              exit 1
            fi
            echo "  ${var,,}: ${val}"
          done

          # Assert country output matches expected (per D-15)
          if [ "$COUNTRY" != "$EXP_COUNTRY" ]; then
            echo "::error::Country mismatch: action returned '${COUNTRY}', expected '${EXP_COUNTRY}'"
            exit 1
          fi
          echo "  ✓ country matches expected: ${COUNTRY}"

          # Workflow-level geo check via ipinfo.io (per D-14 — one provider, plumbing test)
          GEO_COUNTRY="$(curl -fsS https://ipinfo.io/country)"
          if [ "$GEO_COUNTRY" != "$EXP_COUNTRY" ]; then
            echo "::error::Geo check failed: ipinfo.io returned '${GEO_COUNTRY}', expected '${EXP_COUNTRY}'"
            exit 1
          fi
          echo "  ✓ ipinfo.io geo check: ${GEO_COUNTRY}"

          echo "=== All verifications passed ==="

      - name: Disconnect VPN
        uses: ./actions/nordvpn-${{ matrix.region }}/disconnect
        if: always()
```

**5c. drift-issue** — Drift issue management on schedule failures (TEST-08, D-09..D-13):
```yaml
  drift-issue:
    needs: self-test
    if: ${{ failure() && github.event_name == 'schedule' }}
    runs-on: ubuntu-latest
    permissions:
      issues: write
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2
      - name: Create or update drift issue
        env:
          GH_TOKEN: ${{ github.token }}
        run: |
          set -euo pipefail

          # Get list of failed job names from this run
          FAILED_JOBS="$(
            gh run view "${{ github.run_id }}" --json jobs \
              --jq '[.jobs[] | select(.conclusion == "failure") | .name] | join(", ")'
          )"
          RUN_URL="${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}"
          TITLE="region-drift: ${FAILED_JOBS}"
          BODY="Run: ${RUN_URL}\nFailed regions/jobs: ${FAILED_JOBS}"

          # Upsert pattern (D-10): find existing open drift issue
          EXISTING="$(
            gh issue list --label region-drift --state open --json number --jq '.[0].number' 2>/dev/null || true
          )"
          if [ -n "$EXISTING" ]; then
            gh issue comment "$EXISTING" --body "Updated ${TITLE}\n${BODY}"
            echo "Updated existing drift issue #${EXISTING}"
          else
            gh issue create \
              --title "$TITLE" \
              --label region-drift \
              --body "Run: ${RUN_URL}\nFailed regions/jobs: ${FAILED_JOBS}"
            echo "Created new drift issue"
          fi
```
- D-09: One issue per schedule run, lists all failed regions
- D-10: Upsert — find existing open drift issue, create if none
- D-12: Minimal body — run URL + failed region list
- D-13: `gh` CLI everywhere, `GH_TOKEN: ${{ github.token }}` for auth

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fork PR → Workflow Runner | Untrusted fork code triggers the self-test workflow. If the fork guard is bypassed, attacker-controlled code runs with NordVPN secrets in-scope. |
| Workflow Runner → GitHub API (drift issues) | `GITHUB_TOKEN` with `issues: write` can create/modify/close issues. Limited scope — only the current repo's issues. |
| Workflow Runner logs → Public visibility | Logs capture geo-IP check results, action output values, and error messages. Must never echo secret values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | fork-check job | mitigate | Fork-check gate compares `head.repo.full_name` to `github.repository`; matrix job guarded by `if: needs.fork-check.outputs.is_fork == 'false'`. Fork PRs get `::notice::` skip, not a failure. |
| T-04-02 | Tampering | self-test.yml triggers | mitigate | No `pull_request_target` anywhere — enforced by both explicit design and the repo-wide grep guard in actions-lint.yml. The `pull_request` trigger does not expose secrets to fork code. |
| T-04-03 | Information Disclosure | Post-connect verify step logs | mitigate | Post-connect verify only echoes action output values (geo IP, country code, ASN — all non-secret by design). Never reads or echoes `secrets.*` values. Action's own scripts use `set -euo pipefail` (no `set -x`). |
| T-04-04 | Denial of Service | NordVPN session contention | mitigate | Per-region concurrency group (`cancel-in-progress: false`) prevents concurrent runs from fighting over the same VPN session. Runs queue rather than interrupt. |
| T-04-05 | Repudiation | Drift issue management | accept | `GITHUB_TOKEN` acts as `github-actions[bot]`. Acceptable — drift issues are operational alerts, not audit records. |
| T-04-06 | Spoofing | Drift issue creation | mitigate | Drift issue job runs only on `github.event_name == 'schedule'` AND `failure()`. A successful push cannot trigger a drift issue. The label `region-drift` filters issue queries to drift-only. |
</threat_model>
